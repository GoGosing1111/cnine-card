V1125 카드뽑기 실제 서버 지급 확정 패치

[확인된 핵심 원인]
- 서버 상태 확인이 일시 실패하면 클라이언트가 LocalStorage 모드로 전환된 뒤 카드뽑기를 계속 허용했습니다.
- 이 경우 획득 화면과 로컬 도감만 변경되고 실제 D1 user_cards에는 지급되지 않아, 새로고침 후 카드가 사라진 것처럼 보일 수 있었습니다.
- 기존 서버 응답도 user_cards 저장 후 수량을 원자적으로 증명하기 전에 granted:true 결과를 구성했습니다.

[수정]
- 서버 연결이 확인되지 않으면 게임 화면으로 진입하거나 카드뽑기·로컬 계정 생성을 할 수 없도록 차단했습니다.
- 카드 지급, 코인 차감, 조각/마스터의 별, 로그, 요청 영수증을 하나의 D1 batch로 처리합니다.
- 신규 draw_grant_assertions 테이블의 CHECK 제약으로 카드별 실제 보유 수량, 코인, 조각, 마스터의 별, CMS 활성 상태, 완료 영수증이 모두 일치해야 batch가 커밋됩니다.
- 검증 실패 시 전체 지급 batch가 롤백되고 LIMITED 예약 수량도 복구됩니다.
- 각 결과에 지급 전/후 수량과 서버 지급 증명을 포함합니다.
- 클라이언트는 서버 profile의 owned/quantities까지 일치한 경우에만 결과창과 SSR/MA/FUR/LIMITED 획득 연출을 표시합니다.
- 기존 카드 확률, 천장, 크리티컬 효과, 등급별 배경, 결과 UI는 변경하지 않았습니다.

[적용]
- V1124 적용 후 본 패치를 덮어쓰세요.
