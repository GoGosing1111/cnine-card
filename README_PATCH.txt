CNINE PvP 매칭 성능 및 시즌 점수 계산 복구 v9.63

원인:
- 상대 후보 최대 60명을 조회한 뒤 후보마다 userCardScore()를 순차 실행
- 카드 총점 ±15% 조건 때문에 고레벨 후보가 모두 탈락 가능
- 전투 때 공격자·방어자 전체 카드 총점을 다시 조회
- 카드 뽑기 최적화 적용 과정에서 예전 PvP 코드가 다시 포함됨

수정:
- 카드 총점 매칭 조건과 후보별 카드 조회 완전 제거
- 시즌 점수만 사용: ±200 → ±400 → ±700 → ±1200
- 최대 6명이 채워지면 즉시 조회 종료
- 시즌 점수 차이가 가까운 순서로 정렬
- OWNER / ADMIN / 정지 계정 / 본인 제외 유지
- 최근 동일 상대 반복 제한 유지
- 상대 카드에 scoreDiff / expectedWin / expectedLoss 반환
- 전투 덱 2개와 전투 설정을 병렬 조회
- 전투 후 카드 총점 조회 제거
- 시즌 점수 기준 고정 점수표 복구

점수표:
- 상대 +500 이상: 승 +36 / 패 -6
- 상대 +200~499: 승 +30 / 패 -10
- ±199: 승 +24 / 패 -16
- 상대 -200~-499: 승 +18 / 패 -20
- 상대 -500 이하: 승 +12 / 패 -24

변경하지 않음:
- 실제 전투력 계산과 승패 판정
- PvP 에너지
- 코인 보상
- D1 구조와 API 경로
- 카드 뽑기 v9.62 최적화

검증:
- node --check 통과
- ZIP 무결성 검사 통과

적용:
프로젝트 루트에 functions 폴더만 덮어쓴 뒤 배포하세요.
